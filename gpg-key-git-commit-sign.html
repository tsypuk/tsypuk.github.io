Sign git commits with GPG key

Abstraction

Git provides the possibility to add a signature to each commit. By doing this you instrument version control system with instructions to verify the identity of the person who performed commit operation. For this purpose private and public keys need to be generated and configured for git client on local machine to use GPG key. Public key should be submitted to VCS server - github, gitlab etc to verify the validity of commit signature. Here are steps to implement commit signature process.

Generate the key

$ brew install gpg
$ gpg --gen-key
gpg (GnuPG) 2.2.19; Copyright (C) 2019 Free Software Foundation, Inc.
This is free software: you are free to change and redistribute it.
There is NO WARRANTY, to the extent permitted by law.
Note: Use "gpg --full-generate-key" for a full featured key generation dialog.
GnuPG needs to construct a user ID to identify your key.

Enter Real name, email address and passphrase,

Once the process of key generation is complete, it can be verified:

Copy the public key into buffer to paste into GIT account settings.

$ gpg --armor --export E2A9xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx | pbcopy

GPG useful commands

$ gpg --delete-key key-ID
$ gpg --list-keys
$ gpg --delete-key keyID1 keyID2 keyID3
$ gpg --delete-secret-key key-ID
$ gpg --default-new-key-algo rsa4096 --gen-keys

Configure git client to sign commits

gitconfig file should be updated with the following items

Assign signkey the value from gpg command output. Turn on gpg signature by setting gpgsign = true and define the tool that is used for signature. Part of gitconfig with signature enabled:

[user]
    name = XXXXXX
    email = xxxx@xxxx.xxx
    signkey = E2A9xxxxxxxxx
[commit]
    gpgsign = true
[merge]
    tool = opendiff
    conflictstyle = diff3
[gpg]
    program = gpg

Add signature to git-based server

Gitlab: https://docs.gitlab.com/ee/user/project/repository/gpg_signed_commits/

Github: https://help.github.com/en/github/authenticating-to-github/generating-a-new-gpg-key

Commit process

Now once code changes in the repo are completed, on commit operation the message to enter passphrase will appear. On correctly entering the passphrase for GPG key, the commit will be signed.
