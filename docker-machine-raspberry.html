Configure docker machine to work with remote Raspberry PI docker engine

  • ~4 minutes to read

Post Tags: cli docker arm raspberry-pi docker-machine

Abstract

Docker-machine is the utility that allows to provision docker on local host or remote servers and configure local docker client to interact with remote docker engines from local CLI. This post is overview how to use docker machine and setup docker engine on remote server running on Raspberry PI. In this way is possible to configure multiple connections to different docker engines running on public clouds like AWS, Azure or custom DC and easy switch between them from CLI.

Raspberry PI OS name dirty hack

The first thing to do is a dirty hack of manual update ID property with debian value on Raspberry PI, otherwise docker client could not communicate with remote server. This can be done by editing the following file:

$ sudo vim /etc/os-release
PRETTY_NAME="Raspbian GNU/Linux 9 (stretch)"
NAME="Raspbian GNU/Linux"
VERSION_ID="9"
VERSION="9 (stretch)"
ID=debian
ID_LIKE=debian
HOME_URL="http://www.raspbian.org/"
SUPPORT_URL="http://www.raspbian.org/RaspbianForums"
BUG_REPORT_URL="http://www.raspbian.org/RaspbianBugs"

Provision docker with docker machine

If SSH keys are setup for remote server docker machine will use them in process of remote docker installation. Otherwise docker-machine will generate and setup them. Run the following command.

$ docker-machine create --driver generic \
--generic-ip-address 192.x.x.x \
--generic-ssh-key ~/.ssh/raspberrypi_rsa \
--generic-ssh-user pi \
--engine-storage-driver overlay2 \

Connect to remote docker engine

By default, docker client operates with local docker engine, it requires configuration to work with remote docker engine. The following command will output list of ENVs for configuration.

$ docker-machine env raspberry-docker
export DOCKER_TLS_VERIFY="1"
export DOCKER_HOST="tcp://192.x.x.x:2376"
export DOCKER_CERT_PATH="/Users/USER/.docker/machine/machines/raspberry-docker"
export DOCKER_MACHINE_NAME="raspberry-docker"
# Run this command to configure your shell:
# eval $(docker-machine env raspberry-docker)

More easy way to configure docker client - is to run the following command with docker machine name and it will complete auto configuration.

$ eval $(docker-machine env raspberry-docker)

Docker client now is configured and is ready to control docker engine on remote server. But this setup is applied for the current session only. On terminal restart env should be applied again.

Certs regeneration

Certificates can be regenerated for particular remote server on demand:

$ docker-machine regenerate-certs raspberry-docker
