Abstract

Schema evolution is the process of changing payload structure in the data formats by deleting, adding, renaming fields. It is extremely important to handle it with care because incompatible changes can break your consumer or data processing tools. Evolution is more complicated when 3rd parties are involved in data processing and it is not possible to update all consumers, producers at a time.

There are tactics and rules that should be kept in the mind when creating the initial schema and also rules how to create a new version of the schema and make them compatible.

Compatibility types

Two main directions are present in the evolution of compatibility - forward and backward. The term transitive changes identifies that there are several generations of the schema and compatibility is applied to all of them. Full compatibility includes both backward and forward compatibilities.

Forward compatibility test

Adding compatibility tests. Utils class performs write/read cycles with different schema versions involved and performs verification. forwardBroken test verifies that Exception is thrown because V1_2 has incompatible changes.

class ForwardCompatibility {

    final UserV1_2.Builder userV1_2Builder = UserV1_2.newBuilder();
    final UserV3.Builder userV3Builder = UserV3.newBuilder();

    private String fileName = "/tmp/";

    @BeforeEach
    void before(TestInfo testInfo) {
        fileName = "/tmp/" + testInfo.getDisplayName() + ".avro";
    }

    /**
     * Consumer schema X
     * Producer schema X
     */
    @Test
    void sameSchema(TestInfo testInfo) {
        UserV3 userV3Write = userV3Builder
                .setUserName("ALICE")
                .setPhoneNumber("777")
                .setEmail("NONE")
                .setPassword("PASSWORD")
                .setLevel("SE")
                .build();
        writeToAvro(userV3Write, UserV3.class, fileName);
        UserV3 userV3Read = readFromAvro(UserV3.class, fileName);

        assertEquals(userV3Read.getUserName(), "ALICE");
        assertEquals(userV3Read.getPhoneNumber(), "777");
        assertEquals(userV3Read.getEmail(), "NONE");
        assertEquals(userV3Read.getPassword(), "PASSWORD");
        assertEquals(userV3Read.getLevel(), "SE");
    }

    /**
     * Consumer schema X-1
     * Producer schema X
     */
    @Test
    void forward(TestInfo testInfo) {
        UserV3 userV3Write = userV3Builder
                .setUserName("ALICE")
                .setPhoneNumber("777")
                .setEmail("NONE")
                .setPassword("PASSWORD")
                .setLevel("SE")
                .build();
        writeToAvro(userV3Write, UserV3.class, fileName);
        UserV2 userV2Read = readFromAvro(UserV2.class, fileName);

        assertEquals(userV2Read.getUserName(), "ALICE");
        assertEquals(userV2Read.getPhoneNumber(), "777");

        assertEquals(userV2Read.getEmail(), "NONE"); //default email
        assertEquals(userV2Read.getPassword(), "PASSWORD");
    }

    /**
     * Consumer schema X-2
     * Producer schema X
     */
    @Test
    void forwardTransitive(TestInfo testInfo) {
        UserV3 userV3Write = userV3Builder
                .setUserName("ALICE")
                .setPhoneNumber("777")
                .setEmail("NONE")
                .setPassword("PASSWORD")
                .setLevel("SE")
                .build();
        writeToAvro(userV3Write, UserV3.class, fileName);
        UserV1 userV1Read = readFromAvro(UserV1.class, fileName);

        assertEquals(userV1Read.getUserName(), "ALICE");
        assertEquals(userV1Read.getPassword(), "PASSWORD");
        assertEquals(userV1Read.getIsRoot(), false);
    }

    /**
     * Consumer schema X-1
     * Producer schema X but break changes (deleted needed field)
     * <p>
     * org.apache.avro.AvroTypeException: Found io.github.tsypuk.UserV1_2, expecting io.github.tsypuk.UserV1, missing required field password
     */
    @Test
    void forwardBroken(TestInfo testInfo) {
        UserV1_2 userV1_2Write = userV1_2Builder
                .setUserName("ALICE")
                .setPhoneNumber("777")
                .build();
        writeToAvro(userV1_2Write, UserV1_2.class, fileName);

        Assertions.assertThrows(AvroTypeException.class, () -> {
            readFromAvro(UserV1.class, fileName);
        });
    }
}

Backward compatibility test

Adding test for backward compatibility following the same approach

class BackwardCompatibility {

    final UserV1.Builder userV1Builder = UserV1.newBuilder();
    final UserV2.Builder userV2Builder = UserV2.newBuilder();

    private String fileName = "/tmp/";

    @BeforeEach
    void before(TestInfo testInfo) {
        fileName = "/tmp/" + testInfo.getDisplayName() + ".avro";
    }

    @Test
    void sameSchema(TestInfo testInfo) {
        UserV2 userV2Write = userV2Builder
                .setUserName("ALICE")
                .setPhoneNumber("777")
                .setEmail("NONE")
                .setPassword("PASSWORD")
                .build();
        writeToAvro(userV2Write, UserV2.class, fileName);
        UserV2 userV2Read = readFromAvro(UserV2.class, fileName);

        assertEquals(userV2Read.getUserName(), "ALICE");
        assertEquals(userV2Read.getPhoneNumber(), "777");

        assertEquals(userV2Read.getEmail(), "NONE");
        assertEquals(userV2Read.getPassword(), "PASSWORD");
    }

    /**
     * Consumer schema X
     * Producer schema X-1
     */
    @Test
    void backward(TestInfo testInfo) {
        UserV1 userV1Write = userV1Builder
                .setUserName("ALICE")
                .setPassword("PASSWORD")
                .setIsRoot(true)
                .build();
        writeToAvro(userV1Write, UserV1.class, fileName);
        UserV2 userV2Read = readFromAvro(UserV2.class, fileName);

        assertEquals(userV2Read.getUserName(), "ALICE");
        assertEquals(userV2Read.getPhoneNumber(), null);

        assertEquals(userV2Read.getEmail(), "xxx@xx.com"); //default email
        assertEquals(userV2Read.getPassword(), "PASSWORD");
    }

    /**
     * Consumer schema X
     * Producer schema X-2
     */
    @Test
    void backwardTransitive(TestInfo testInfo) {
        UserV1 userV1Write = userV1Builder
                .setUserName("ALICE")
                .setPassword("PASSWORD")
                .setIsRoot(true)
                .build();
        writeToAvro(userV1Write, UserV1.class, fileName);
        UserV3 userV3Read = readFromAvro(UserV3.class, fileName);

        assertEquals(userV3Read.getUserName(), "ALICE");
        assertEquals(userV3Read.getPhoneNumber(), null);

        assertEquals(userV3Read.getEmail(), "xxx@xx.com"); //default email
        assertEquals(userV3Read.getPassword(), "PASSWORD");
        assertEquals(userV3Read.getLevel(), "NOT_SET");
    }

    /**
     * Consumer schema X
     * Producer schema X-1 but break changes (added non optional field with non default values)
     * <p>
     * org.apache.avro.AvroTypeException: Found io.github.tsypuk.UserV1, expecting io.github.tsypuk.UserV1_2, missing required field phone_number
     */
    @Test
    void backwardBroken(TestInfo testInfo) {
        UserV1 userV1Write = userV1Builder
                .setUserName("ALICE")
                .setPassword("PASSWORD")
                .setIsRoot(true)
                .build();
        writeToAvro(userV1Write, UserV1.class, fileName);

        Assertions.assertThrows(AvroTypeException.class, () -> {
            readFromAvro(UserV1_2.class, fileName);
        });
    }
}

Schema Evolution best practices

  • Provide default values to all fields that could change in the next schema versions.

  • Do not rename fields, if needed use aliases.

  • In next schemas, always give default values and do not delete required fields

  • Apply default values to enums, it will be used during resolution when the reader encounters a symbol from the writer that isn’t defined in the reader’s schema.
